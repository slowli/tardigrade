Task management in workflows.

Similarly to other async runtimes, a task is a self-contained unit of work with the 'static lifetime. Tasks are spawned from the workflow code using spawn() or try_spawn() functions, which return the JoinHandle for the created task. This handle can be polled to completion, or be used to abort the task.

Error handling

Errors during task execution are encapsulated in TaskError, which provides a way to track source code location and the cause of errors. Unlike with threads or tasks in most other async runtimes, a panic in any task is unrecoverable. Depending on the host settings, it may lead to the erroneous workflow termination, or to its rollback.
