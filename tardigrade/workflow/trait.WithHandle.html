Expand description

Type with a polymorphic handle depending on a HandleFormat.

Conceptually, a handle collects simplest Handles (that for receivers and senders) into more complex structures. A handle can be taken from an untyped collection of handles (e.g., UntypedHandles) or, conversely, inserted into an untyped collection.

A handle needs to be defined for a workflow in order to specify which handles it receives on initialization. The intended way to do this is to define a dedicated handle type with the help of the WithHandle derive macro. If necessary, its implementation can be delegated from the (usually empty) workflow type. See the workflow module docs for an example.
