Expand description

Read-write connection to the worker storage.

This view is not required to be transactional, although it can be if the underlying storage supports this.

Releases the connection back to the pool.

  • If this connection is transactional, atomically commits changes to the storage.
  • If this connection is not transactional, the changes are expected to take effect immediately, and this method can be no-op.
